## Critical Rules

### Rule 1: Additional Other Expenses goes on Line 17, NOT Guaranteed Payments
On Form 8825, the detail worksheet for miscellaneous expenses (pest control, office supplies, cleaning, etc.) MUST be attached to **"Additional Other Expenses" on line 17**. Never attach it to Guaranteed Payments on line 16. This mistake cost 40 minutes on the first return.

### Rule 2: Every 4562 asset needs "For: 8825"
The "For:" field on each depreciation asset tells Drake which form receives the depreciation. For rental real estate, it must be `8825`. If left blank, depreciation won't flow to Form 8825 line 14 and income will be overstated.

### Rule 3: Fill ALL blue fields on first pass
Common blue fields people miss: K-1 Country of Citizenship → US, K-1 Max percentage owned, PIN SSN, M-1 Tax-Basis checkbox, 8825 QBI "trade or business" → Y.

### Rule 4: Never restart from scratch
If you find errors after calculating, fix them in place. Restarting wastes all prior work. Every error in Drake can be fixed by navigating to the right screen.

### Rule 5: One calculate cycle
If rules 1-4 are followed, you should only need to calculate once.

## Investment Partnership / Multi-Tier K-1P Pattern (NEW — LRA Ventures + LRA Legacy, April 2026)

Based on completing LRA Ventures LLC (EIN 86-3842853) and LRA Legacy LLC (EIN 86-3706837) — investment holding partnerships with pass-through K-1 income from multiple hedge funds, PE funds, and a JPM brokerage account. 50/50 allocation to Tarek & Lina Hammoud. FL resident state. Multi-tier structure: Ventures (80% Endeavors, 20% Legacy) holds fund investments; Legacy directly owns interests in multiple funds + JPM brokerage + receives 20% pass-through from Ventures.

### Key Differences from Rental 1065s

1. **No Form 8825 for rental income entered directly** — Rental flows in via K1P screens from lower-tier partnerships (e.g., Sharp Key Coral Palms). Drake's K1P Box 2 (rental RE income) auto-populates Form 8825.
2. **K1P screen is the primary data entry screen** — one K1P per fund/entity K-1 received. Navigate: "PTR K, K-1, K-2, K-3" tab → Screen Menu → "K1P: Schedule K-1 from Another Partnership."
3. **K screen (Schedule K)** has ± adjustment fields — these are ADDED to K1P computed totals. Used for recon adjustments (reclass items between Box 11/13 sub-codes). The K screen has Income, Deductions, Credits, and AMT tabs.
4. **K screen "Other" links on ZZ sub-codes** open Detail Worksheets with Description/Amount columns for Line 11 ZZ, Line 13 ZZ, Line 20 ZZ items.
5. **Form 6781** (Section 1256 Contracts) — auto-generated from K1P Box 11 Code C entries. Verify the 60/40 ST/LT split populates correctly on Schedule D.
6. **Schedule D** — auto-populated from K1P Box 8 (ST) and Box 9a (LT) plus Form 6781 flow-through. Do NOT enter capital gains directly on Schedule D.
7. **QBI via K199 screen** — For activities reporting Section 199A data (Box 20 Code Z), enter via K199 screen. CRITICAL: each QBI-eligible activity needs its own Statement A entry. Rental QBI from lower tiers (e.g., Sharp Key) must be entered as SEPARATE trade/business lines with rental income, W-2 wages, and UBIA.
8. **Partnership Representative (B4 screen)** — Required for 2025+ returns. Enter designated individual name, SSN, address, phone. Missing = EF Error 933/935.
9. **No depreciation entered directly** — All depreciation is embedded in fund K-1 pass-throughs.
10. **Item M on K-1 = "No"** — Partner did NOT contribute property with built-in gain/loss (check on each K1 screen).

### K Screen ± Fields (Schedule K Adjustments)

The K screen shows computed totals from all K1Ps. The ± fields BELOW each line allow manual adjustments (positive = add, negative = subtract). These are used for:
- Reclassifying Box 11 sub-items between portfolio income, Section 1256, and Other
- Reclassifying Box 13 sub-items between investment interest expense and other deductions
- Adding JPM brokerage income (interest, dividends) that doesn't come via K1P
- Recon adjustments from the workpaper

**DANGER:** Do NOT accidentally type into a ± field when navigating. If you type "K1" into a ± field, it opens an empty Detail Worksheet. Click "Exit Without Saving" immediately.

### QBI / K199 Screen — Section 199A

**Navigation:** Type "K199" in search or go to PTR K tab → K199.

Each QBI-eligible activity needs a SEPARATE entry:
- **Trade/business activities** (Box 1 OBI) → enter with "Other Income" and "Other Deductions" from recon adjustments
- **Rental activities from lower tiers** (e.g., Sharp Key) → enter as separate activity with:
  - Rental Income (Loss) = the Box 2 amount from that fund's K-1
  - W-2 Wages = from that fund's Box 20 Code Z
  - UBIA = from that fund's Box 20 Code Z

**CRITICAL AUDIT FINDING (LRA Legacy):** The QBI Statement A was missing Sharp Key's rental QBI data entirely ($710,967 rental / $3,638 W-2 / $2,576,662 UBIA). This WILL break the QBI deduction on the 1040. Always cross-check every lower-tier K-1 Box 20 Code Z against the QBI Statement A output.

### Partnership Representative (B4 Screen)

**Navigation:** Type "B4" in search.

Required fields for 2025+:
- Partnership Representative name
- SSN or EIN of designated individual
- Address (street, city, state, ZIP)
- Phone number (use 847-922-5680 for Hammoud family)
- Check "PTR REP" checkbox on the K-1 screen for the designated partner

Missing any of these = EF Error 933 (invalid/missing partnership rep) and EF Error 935 ×4 (missing/invalid PR address).

### Audit Checklist — Investment Partnership 1065

After calculating and before declaring the return done, verify:

1. **Schedule K totals** match recon workpaper line by line (K-1 through K-14c)
2. **QBI Statement A** has ALL trade/business activities AND all rental activities with QBI data (W-2 wages, UBIA)
3. **Box 19/20 items** from EVERY fund K-1 are entered — check for:
   - Gross receipts for 448(c) (Code AG)
   - Excess business interest income (Code AF)
   - Section 704(c) information
   - Investment income/expenses (Codes A/B)
4. **Schedule L** is either populated OR elected out via Sch B Item 4
5. **Beginning capital accounts** match prior year ending balances (link proforma)
6. **K-1 allocations** to each partner = correct % of Schedule K totals
7. **Form 6781** Sec 1256 total = sum of all K1P Box 11 Code C entries
8. **Form 8825** rental total = sum of all K1P Box 2 entries
9. **Partnership Representative** address fully populated on B4 screen
10. **K-3 exemption** — verify it's appropriate if any lower-tier entity provided a K-3
