## Critical Rules

### Rule 1: Use Heads Down Entry (Ctrl+N) for ALL 8867 checkboxes
Form 8867 checkboxes are extremely finicky with direct mouse clicks. Do NOT waste time trying different coordinates. Immediately press Ctrl+N to enter Heads Down Entry mode, which numbers every field. Type the field number + Enter to navigate, then X + Enter to check a checkbox. This rule alone would have saved 60 minutes on the first return.

### Rule 2: Check K-1 QBI MFC after every K-1 import
When K-1s are exported from partnership/S-Corp returns to a 1040, the QBI (Qualified Business Income) entries are auto-created but the MFC (Multi-Form Code) field may be blank. After importing K-1s:
1. Navigate to the K-1 QBI summary screen
2. Check every row — if MFC is blank, set it to `1`
3. Drake shows a warning dialog about missing fields but saves anyway — don't ignore it

### Rule 3: Q7a on Form 8867 — Leave BLANK for most clients
Q7a asks about Form 8862 (recertification after prior EIC/CTC denial). Answering "Yes" tells Drake the client needs Form 8862, triggering EF error 5798. For most clients who were NOT previously denied these credits, leave Q7a blank. Do NOT set it to Yes or N/A.

### Rule 4: Batch all 8867 answers in one pass
Form 8867 has ~12 questions across two sections. Answer ALL of them in a single Heads Down Entry session before calculating. Iterating (answer one → calculate → find next error → answer → calculate) wastes 5 minutes per cycle.

### Rule 5: Fill ALL blue fields on first pass
Blue fields = required for e-filing. Missing them means EF errors that waste calculate-fix cycles. Common blue fields on 1040:

**Screen 1:** SSN, First Name, Last Name, Address, City, State, ZIP, Filing Status, DOB
**Screen 2:** Spouse SSN, First Name, Last Name, DOB (if MFJ)
**Dependents:** SSN, First Name, Last Name, Relationship, DOB, Months in home
**Schedule C:** Business code, Business name, Address, City, State, ZIP, Accounting method, EIN (if applicable)
**99N:** Payer name, Payer address, Amount, Link to Schedule C
**PIN:** ERO PIN, Taxpayer PIN, Phone number
**1099-DIV:** Payer name, Ordinary dividends, Qualified dividends

### Rule 6: 99N (1099-NEC) flows automatically to Schedule C — DO NOT double-count
When a 1099-NEC is entered on the **99N screen** and linked to a Schedule C, Drake automatically flows that 1099-NEC amount into Schedule C gross receipts. If the client's P&L total gross income INCLUDES the 1099-NEC amount (which it almost always does), then Schedule C line 1 "Gross receipts and sales" must contain ONLY the non-1099 portion:

**Formula:** Schedule C Line 1 = Total P&L Gross Income − Sum of all linked 1099-NEC amounts

**Example (Dvorak):**
- Total P&L Gross Income: $138,832
- 1099-NEC from First American Title: $103,918 (on 99N screen, linked to Schedule C)
- Schedule C Line 1 should be: $138,832 − $103,918 = **$34,914**
- Drake then calculates: $34,914 (manual) + $103,918 (99N flow) = $138,832 ✓

**What goes wrong if you enter the full P&L amount:** Drake shows $138,832 + $103,918 = $242,750 in gross receipts — nearly double the correct income. This inflates total income, SE tax, and balance due dramatically.

### Rule 7: Fill Dependent Due Diligence tab on FIRST pass
The dependent screen has TWO tabs: "Dependent" (basic info) and "Due Diligence" (required for CTC/ODC). Both must be filled on the first visit — don't wait to discover it from an EF error. See the Dependent Due Diligence section below for what to answer.

### Rule 8: Form 8867 Q7 must be "Yes"
Q7 asks "Did you ask the taxpayer if any credits were disallowed or reduced in a previous year?" — the answer is always **Yes** (because we asked). "No" means we failed to ask, which triggers EF error 4906. This is different from Q7a (which should be BLANK).

### Rule 9: Always review prior year return thoroughly
Cross-reference the prior year (PY) return for: estimated payments, depreciation schedules, NOL carryforwards, Schedule E rental properties, recurring income sources. The PY return is the best checklist for what the current year return needs.

### Rule 10: Review ALL pages of ALL uploaded documents
Never skip pages regardless of file size. Tax documents can have critical data on any page — 1099s, K-1s, or supporting schedules can appear anywhere in the workpaper.

## Screen Details — Coordinate Maps and Batch Sequences

These coordinates are approximate and based on Drake running at standard resolution. Always take a screenshot to verify before batch-clicking. The key insight: **take ONE screenshot at the start of a screen, build your entire batch sequence from that screenshot, execute the batch, then take ONE verification screenshot.**

**Two tabs — FILL BOTH:**
1. **"Dependent" tab** — basic info (name, SSN, relationship, DOB, months)
2. **"Due Diligence" tab** — required for CTC/ODC eligibility

**Dependent tab fields:**

| Field | Blue? | Notes |
|-------|-------|-------|
| First Name | Yes | |
| M.I. | No | |
| Last Name | Yes | Will default to taxpayer's last name if `=` entered |
| SSN | Yes | |
| Relationship | Yes | Dropdown: SON, DAUGHTER, STEPSON, etc. |
| Months in home | Yes | 1-12 |
| Date of Birth | Yes | MM-DD-YYYY |

**Due Diligence tab — Answer patterns (for a qualifying child who lives with taxpayer):**

| Question | Answer | Why |
|----------|--------|-----|
| Q1: Is dependent unmarried OR married and can be claimed? | **Yes** | Minor child living with parent |
| Q2: Did dependent live with taxpayer in U.S. > half year? | **Yes** | 12 months in home |
| Q3: If not son/daughter, did you ask why parent not claiming? | **Not Applicable** | IS the taxpayer's child |
| Q4: Could another person qualify to claim this dependent? | **No** | Custodial parent |
| Q10: Qualifying person is citizen/national/resident? | **Yes** | U.S. citizen |
| Q11: Did you explain they may not claim if child didn't live with them > half year? | **Yes** | Due diligence requirement |
| Q12: Did you explain rules about CTC/ODC for divorced/separated parents? | **Yes** | Due diligence requirement |

**Checkbox precision warning:** The Yes/No/Not Applicable checkboxes are very close together (~25px apart). If a click doesn't register:
1. Try clicking the "Yes" checkbox first (leftmost) — this column is most reliable
2. Use Tab + Space to navigate between Yes/No/N/A
3. Zoom in (mcp__computer-use__zoom) to verify which column is actually checked
4. Q4 "No" specifically is hard to click — after checking Q4 "Yes" first, use Tab→Space to move to "No"

**Also check at least one residency document checkbox** under "Residency of Qualifying Child" — e.g., "Did not rely on any documents, made notes in file"

### Form 8867 — Due Diligence Checklist

**Navigation:** Type `8867` in screen search.

**THIS IS THE HARDEST SCREEN ON 1040.** The checkboxes are notoriously unresponsive to direct mouse clicking via computer-use.

**MANDATORY: Use Heads Down Entry mode (Ctrl+N)**

1. Navigate to Form 8867 screen
2. Press Ctrl+N (or right-click → Heads Down Entry)
3. Every field gets a number displayed
4. For each question, type the field number + Enter to navigate
5. Type X + Enter to check the Yes/No/N/A checkbox
6. Press Escape when done

**Answer patterns for typical returns:**

| Question | Topic | Typical Answer | Notes |
|----------|-------|----------------|-------|
| Q1 | Complete return based on info? | **Yes** | Always Yes |
| Q3 | Satisfy knowledge requirement? | **Yes** | Always Yes for proper preparation |
| Q4 | Info appear incorrect/inconsistent? | **No** | No if everything checks out |
| Q5 | Satisfy record retention? | **Yes** | Always Yes |
| Q6 | Ask taxpayer about documentation? | **Yes** | Always Yes — 4906 if missing |
| Q7 | Ask about prior disallowed credits? | **Yes** | **MUST be Yes — 4906 error if No** |
| **Q7a** | **Form 8862 recertification** | **BLANK** | **DO NOT check Yes — triggers 5798** |
| Q8 | Schedule C due diligence | **Yes** | Must be Yes if Schedule C exists — 4906 if N/A |
| Q14 | HOH — qualifying home? | **Yes** | If filing HOH |
| Q15 | All answers true and correct? | **Yes** | Always Yes |

**Bottom section questions** (Was taxpayer nonresident alien? Could taxpayer be qualifying child? Main home in U.S.? etc.) — answer based on taxpayer's actual situation. For most U.S. resident filers: "Was nonresident?" → No, "Main home in U.S.?" → Yes, etc.

**Coordinate map (fallback if Heads Down Entry fails):**
- Main questions: Yes≈x=979, No≈x=1003, N/A≈x=1027
- Q6 Yes: (979, 450)
- Q7 Yes: (979, 472) — **VERIFY this is actually in the Yes column, not No**
- Q8 Yes: (979, 525)
- Bottom section: Yes≈x=982, No≈x=1009
- **WARNING:** Main questions and bottom section use DIFFERENT x-coordinates

## Common 1040 EF Errors

| Code | Issue | Fix |
|------|-------|-----|
| 4906 | Due diligence question not answered Yes | Check 8867 Q6, Q7, Q8 — most common are Q7 (must be Yes, not No) and Q8 (must be Yes if Schedule C) |
| 4524 | Dependent due diligence missing | Fill Dependent Due Diligence tab — answer ALL questions Q1-Q4, Q10-Q12 |
| 4912 | Screen 2 due diligence not answered | Dependent Due Diligence tab needs Yes/No answers |
| 4922 | Incomplete dependent information | Fill all fields on Dependent Due Diligence tab |
| 4951 | 8867 bottom section incomplete | Answer all bottom-section checkboxes |
| 5798 | Form 8862 recertification required | Q7a was set to Yes — uncheck it (leave blank) |
| REQUIRED | Missing MFC on K-1 QBI | Open K-1 QBI summary, set MFC=1 on blank rows |
| REQUIRED | Missing SSN or DOB | Check Screen 1/2, dependents |
| IL 9320 | IL SSN/auth missing | Add SSN on PIN + check IL Authorization |

## K-1 QBI — Detailed Notes

When K-1s are imported from pass-through entities, Drake creates QBI entries on the K-1 QBI summary screen. Key fields:

| Field | Description | Required? |
|-------|-------------|-----------|
| Source | K1P (partnership) or K1S (S-Corp) | Auto |
| EIN | Entity EIN | Auto |
| Entity Name | Partnership/S-Corp name | Auto |
| MFC | Multi-Form Code | **YES — often blank, must set to 1** |
| QBI Amount | Qualified business income | Auto from K-1 |
| W-2 Wages | Entity W-2 wages | Auto from K-1 |
| UBIA | Unadjusted basis of assets | Auto from K-1 |

**The MFC field is the #1 issue.** Drake shows a dialog warning that fields are missing but still saves. Always verify MFC after K-1 import.

## Speed Improvement Checklist

Each return should be faster than the last. The main time sinks and their fixes:

| Time Sink | Typical Minutes Lost | Fix |
|-----------|---------------------|-----|
| Guessing field coordinates | 10-30 min | Use the coordinate maps above |
| Missing blue fields → EF error loop | 15-30 min | Fill ALL blue fields on first pass (Rule 5) |
| 8867 checkbox clicking | 10-30 min | Always use Heads Down Entry (Rule 1) |
| 99N double-counting Schedule C | 30-60 min | Subtract 99N from P&L total (Rule 6) |
| Forgetting dependent due diligence | 15-30 min | Fill both tabs on first visit (Rule 7) |
| Not reading PY return | 30-60 min | Always cross-reference PY (Rule 9) |
| Context window exhaustion | 30-60 min | Batch more, screenshot less, work faster |

**After every return:** Update this file with new coordinate data, new EF errors, and new batch sequences discovered. Push to GitHub. The goal is that return #5 takes under 15 minutes.
